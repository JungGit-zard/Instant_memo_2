# 매일 할 일 메모앱 — 개발 핸드오프 문서

작성일: 2026-06-12
산출물: `daily-todo.jsx` (React 단일 파일), `모바일앱-전환-준비-투두리스트.md`

---

## 1. 프로젝트 개요

날짜별로 할 일을 메모·관리하는 투두 앱. Claude 아티팩트 환경에서 동작하는 React 프로토타입으로 개발됨. 최종 목표는 모바일 앱(iOS/Android) + 로컬 PC 버전 + 기기 간 데이터 동기화.

## 2. 구현된 기능 목록

### 기본 기능
- 할 일 추가 (Enter 지원, 빈 입력 방지, 추가 후 입력창 포커스 유지로 연속 입력)
- 체크/해제 (체크 시 0.55초 머문 뒤 완료 섹션으로 이동, 그 사이 다시 누르면 취소)
- 더블클릭 인라인 수정 (Enter 저장 / Esc 취소 / blur 저장)
- 삭제 + 5초간 "되돌리기" 토스트 (id 중복 복원 가드 포함)
- 완료 항목 일괄 삭제 (되돌리기 가능)
- 필터: 전체 / 진행중 / 완료
- 진행률 바, 개수 요약, 빈 목록 안내, 완료 섹션 접기/펼치기

### 날짜 기능
- 날짜별 독립 목록 (‹ › 버튼으로 하루씩 이동, "오늘" 버튼)
- **달력 모달**: 헤더 날짜 블록 클릭 → 년(« »)/월(‹ ›) 이동, 날짜 탭으로 점프. 미래 날짜 예약 등록 가능
- **자동 영구 이월**: 오늘 화면을 열면 마지막 이월일 이후 날짜를 스캔해 미완료 항목을 오늘로 "이동"(원본에서 제거). 체크 전까지 매일 따라옴. 항목에 "M/D부터" 배지 표시. 첫 스캔은 최대 366일 제한

### 조작 기능
- 드래그 앤 드롭 순서 재배치 (⠿ 핸들, 진행중 항목만)

### 저장
- `window.storage` (아티팩트 전용 key-value API)에 날짜별 자동 저장
- 키 구조: `todos:YYYY-MM-DD` (항목 배열 JSON), `meta:lastRollover` (마지막 이월일)
- 항목 스키마: `{ id, text, done, createdAt, carried?, origin? }`

## 5. 환경 제약 사항 (이식 시 필수 확인)

- `window.storage`는 **아티팩트 전용 API** — 다른 환경 이식 시 1순위 교체 대상 (어댑터 계층 도입 권장)
- localStorage/sessionStorage는 아티팩트에서 사용 불가 (사용하지 않았음)
- 푸시 알림은 현 환경에서 불가. 네이티브 전환 시: Android 사실상 무제한, iOS 로컬 알림 대기 64개 제한
- 드래그 높이 계산은 행 높이 균일을 가정 (여러 줄 항목이 섞이면 미세 오차 가능)
- 저장소 호출 횟수 제한 → 첫 이월 스캔 366일 캡, 366일 이상 미접속 시 그 이전 미완료 항목은 해당 날짜에 잔류

## 8. 데스크탑 IDE에서 실행하는 방법

`daily-todo.jsx`는 단일 React 컴포넌트라 Vite 프로젝트에 넣으면 바로 돌아간다:

```bash
npm create vite@latest daily-todo -- --template react
cd daily-todo && npm install
# daily-todo.jsx 를 src/App.jsx 로 교체 (export default 그대로 사용)
npm run dev
```

**주의 — 반드시 먼저 처리할 것:**
- `window.storage`는 아티팩트 전용 API라 IDE 환경에는 없음 → 실행 전에 임시 어댑터를 추가할 것:
```js
// src/storage-shim.js — 개발용 localStorage 어댑터 (main.jsx 최상단에서 import)
if (!window.storage) {
  window.storage = {
    async get(key) {
      const v = localStorage.getItem(key);
      if (v === null) throw new Error("not found");
      return { key, value: v };
    },
    async set(key, value) { localStorage.setItem(key, value); return { key, value }; },
    async delete(key) { localStorage.removeItem(key); return { key, deleted: true }; },
    async list(prefix = "") {
      return { keys: Object.keys(localStorage).filter(k => k.startsWith(prefix)) };
    },
  };
}
```
- 위 shim은 개발 임시용. 정식으로는 투두리스트 문서 2장의 스토리지 어댑터 계층으로 교체할 것 (PC: IndexedDB / 모바일: SQLite)
- Tailwind는 사용하지 않았고 전부 인라인 스타일이라 추가 설정 불필요

## 9. Firebase Realtime Database 동기화 패치 (적용됨)

PC↔모바일 실시간 데이터 공유용. 코드 상단의 `FIREBASE_CONFIG`와 `SYNC_ID`를 채우면 켜지고, **비워두면 기존 로컬 전용 모드로 동작**한다 (기능 저하 없음).

### 켜는 방법
1. Firebase 콘솔 → 프로젝트 생성 → 빌드 → **Realtime Database** 만들기 (Firestore 아님 주의)
2. 규칙 탭: `{"rules":{"rooms":{"$room":{".read":true,".write":true}}}}`
3. 프로젝트 설정 → 웹 앱 추가 → 구성값을 `FIREBASE_CONFIG`에 붙여넣기 (`databaseURL` 필수)
4. `SYNC_ID`에 긴 무작위 코드 지정, **모든 기기에 같은 값** 입력

### 동작 방식
- 로컬 저장(항상) + 원격 푸시(saveChain 경유), 현재 보는 날짜 노드를 `on("value")` 실시간 구독
- 자기 에코는 `lastPushed` payload 비교로 무시 (무한 루프 방지)
- 이월 시 원본 날짜 정리 결과·병합 결과를 원격에도 반영, `meta/lastRollover`를 원격과 비교해 기기 간 중복 이월 방지
- 데이터 경로: `rooms/{SYNC_ID}/days/{YYYY-MM-DD}` = `{items:[...], updatedAt}`

### 한계와 주의 (알고 쓸 것)
- **보안**: 위 MVP 규칙은 SYNC_ID를 아는 누구나 접근 가능. 정식 서비스 전 Firebase Auth 규칙으로 교체 필수
- **충돌 해결이 "하루 노드 단위 최신 우선"**: 두 기기가 같은 날짜를 동시에(특히 오프라인에서) 편집하면 늦게 접속한 쪽이 통째로 덮을 수 있음 → 항목 단위 병합(투두리스트 문서 5-3의 tombstone)으로 개선 예정
- **아티팩트 미리보기에서는 외부 네트워크 제한(CSP)으로 원격 연결이 막힐 수 있음** → 동기화 테스트는 IDE(Vite) 환경에서 할 것. config를 비워두면 미리보기는 정상 동작
- SDK는 cdnjs에서 compat 빌드를 동적 로드 (`window.firebase` 전역 사용). Vite로 옮긴 뒤에는 `npm i firebase` + 모듈 import 방식으로 교체해도 됨
